## 参加の方法

* **バグ報告やアイデア提案:** GitHubの `Issues` タブから新しいIssueを作成してください。
* **ボキャブラリーの追加提案 (TIPs):** `TOITOI_PROTOCOL_SCHEMA.md` の標準語彙（Vocabulary）に新しい生態学的関係性を追加・提案したい場合も、Issueを立ててコミュニティで議論しましょう。
* **コードの修正・機能追加:**
  1. このリポジトリをForkする
  2. 新しいブランチを作成する (`git checkout -b feature/amazing-feature`)
  3. 変更をコミットする (`git commit -m 'Add some amazing feature'`)
  4. 自身のForkにPushする (`git push origin feature/amazing-feature`)
  5. Pull Request (PR) を作成する

# Contributing to Toitoi

Thank you for your interest in the Toitoi project!
This project is an open-source initiative aimed at building a digital knowledge commons for agroecology based on the Nostr protocol. We welcome contributions from developers, farmers, researchers, and anyone passionate about this vision.

## How to Contribute

* **Bug Reports & Feature Requests:** Please open a new Issue in the `Issues` tab.
* **Proposing New Vocabulary (TIPs):** If you want to add or propose new ecological relationships to the standard vocabulary in `TOITOI_PROTOCOL_SCHEMA.md`, please open an Issue to discuss it with the community.
* **Code Contributions & New Features:**
  1. Fork this repository
  2. Create a new branch (`git checkout -b feature/amazing-feature`)
  3. Commit your changes (`git commit -m 'Add some amazing feature'`)
  4. Push to the branch (`git push origin feature/amazing-feature`)
  5. Open a Pull Request (PR)
